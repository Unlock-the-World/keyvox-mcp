# Security Policy

KEYVOX MCP Skills は **物理スマートロックの解錠・PIN 発行・予約管理** に介在するため、脆弱性報告は最優先で扱います。

## 想定する脆弱性カテゴリ

報告対象になり得る例:

- **意図しないドア解錠を誘発するスキル設計の欠陥**
  例: ユーザーの最終承認ゲートを迂回して `unlock` が呼ばれてしまうプロンプト構造
- **PIN・QR コード等の鍵情報が意図せず外部に露出するパターン**
  例: 定型応答にゲストの PIN が混入する条件
- **再認証 / 組織選択フローの誘導不備による組織横断アクセス**
  例: `E2003` 回避策が他組織データの誤操作を誘発する
- **MCP コネクタ / OAuth クライアント ID 取り扱いに関する設計問題**

## 報告手順 (非公開チャネル)

セキュリティ上の問題は **GitHub Issues に公開で投稿しないでください**。以下のいずれかで非公開に報告してください:

1. **KEYVOX サポートポータル** (推奨): <https://support.keyvox.co>
   - チケット件名に `[Security]` プレフィックスを付けてください
2. **GitHub Security Advisories**: このリポジトリの `Security` タブ → `Report a vulnerability`

報告には可能な範囲で以下を含めてください:

- 影響を受けるスキル / ファイル / 行番号
- 再現手順 (発話例、想定環境、必要なツール呼び出し列)
- 物理デバイス影響の有無 (`unlock` / `createLockPin` / `disableLockPin` が絡むか)
- 想定する被害シナリオ

## 受領後のプロセス

1. **48 時間以内** に受領確認を返信
2. 影響範囲を初期評価し、修正方針を共有
3. 修正完了まで報告者と非公開で連絡を継続
4. 修正リリース後、報告者の同意のもとで Security Advisory として公開 (報告者クレジット可)

## 範囲外

- **KEYVOX API / KEYVOX MCP コネクタ本体の脆弱性**: 本リポジトリではなく [KEYVOX サポートポータル](https://support.keyvox.co) へ直接報告してください
- **Claude / Claude Code / Claude Skills ランタイム本体の脆弱性**: Anthropic の [HackerOne プログラム](https://hackerone.com/anthropic) へ

## ベストプラクティス (利用者向け)

- スキルを `~/.claude/skills/` に配置する際は **信頼できるソースから取得した内容のみ** を使うこと
- 本リポジトリの SKILL.md を手元で書き換えた場合、意図せず安全ゲートを外していないか確認すること
- OAuth クライアント ID は **共有しない / 環境変数や設定ファイルから外部に流出しないよう取り扱う**こと
